# WebChat — նախագծի արձանագրություն

Այս փաստաթուղթը նկարագրում է WebChat նախագիծը, դրա տեղադրումը և շահագործումը։ Անգլերեն տարբերակի համար տե՛ս [README.md](../README.md)։

---

## Ներածություն

WebChat-ը պարզ **PHP + MySQL** վրա հիմնված չաթ է, որտեղ նոր հաղորդագրությունները բրաուզերում թարմացվում են **կարճ ընդմիջումներով հարցումների (polling)** միջոցով։ Միջերեսի տեքստերը հայերեն են։

---

## Պահանջներ

- **PHP** 7.4 կամ ավելի նոր (խորհուրդ է տրվում 8.x), ընդլայնումներով՝ `mysqli`, `session`, ինչպես նաև `fileinfo` (ֆայլերի վերբեռնման համար)։
- **MySQL** 5.7+ կամ **MariaDB** 10.3+։
- Վեբ սերվեր, որը կատարում է PHP (օր. Apache + `mod_php`, կամ Nginx + PHP-FPM), կամ **միայն տեղական փորձարկման** համար՝ PHP-ի ներդրված սերվերը։

---

## Տեղադրում

### 1. Document root (վեբ արմատային թղթապանակ)

Վեբ սերվերի **document root**-ը պետք է ցույց տա այս նախագծի թղթապանակը (որտեղ գտնվում են `index.php`, `assets/`, `php/` և այլն), որպեսզի աշխատեն հետևյալ հասցեները՝

- `/index.php`, `/login.php`, `/users.php`, `/chat.php`
- `/assets/css/style.css`, `/assets/js/...`
- `/php/signup.php`, `/php/login.php` և այլն

**Տեղական փորձարկում PHP-ի ներդրված սերվերով** (միայն մշակում, ոչ թե արտադրություն)՝

```bash
cd /path/to/WebChat-main
php -S localhost:8080
```

Բացեք `http://localhost:8080/index.php`։

### 2. Տվյալների բազա

Ստեղծեք բազան և աղյուսակները `sql/schema.sql` ֆայլով (անունը փոխելու դեպքում համապատասխանեցրեք `php/config.php`-ին կամ `config.local.php`-ին)՝

```bash
mysql -u root -p < sql/schema.sql
```

Կարելի է ներմուծել նաև phpMyAdmin կամ այլ գործիքով։

### 3. Գրելու իրավունք վերբեռնվող թղթապանակներում

Վեբ սերվերի օգտվողը (օր. `www-data`) պետք է կարողանա **գրել** հետևյալ թղթապանակներում՝

- `php/images/` — գրանցման նկարներ, `profile.png` լռելյայն նկարը
- `php/uploadedFiles/` — չաթի կցորդ ֆայլեր

Օրինակ macOS/Linux-ում (ուղիներն ու օգտվողը կարող են տարբեր լինել)՝

```bash
chmod -R ug+rwX php/images php/uploadedFiles
# Անհրաժեշտության դեպքում՝
# sudo chown -R www-data:www-data php/images php/uploadedFiles
```

### 4. Կարգավորում (կոնֆիգուրացիա)

Լռելյայն կապը MySQL-ի հետ նկարագրված է `php/config.php` ֆայլում (`localhost`, `root`, դատարկ գաղտնաբառ, բազա `chat`)։

**Տեղական գաղտնաբառեր և հոստեր չպահելու համար** (առանց `config.php`-ը փոփոխելու պահեստում)՝

1. Պատճենեք `php/config.local.example.php` → `php/config.local.php` (այս ֆայլը `.gitignore`-ում է, commit չի լինի)։
2. Խմբագրեք `config.local.php`-ում `$db_host`, `$db_user`, `$db_pass`, `$db_name`։

`php/config.php`-ն ավտոմատ կկարդա `php/config.local.php`-ը, եթե այն գոյություն ունի։

### 5. Անվտանգություն — CSRF թոքեն

Գրանցման, մուտքի և չաթի ձևերում կա **սեսիայի հետ կապված CSRF թոքեն**։ Ստորև բերված PHP վերջնակետերը ստուգում են այն՝

- `php/signup.php`, `php/login.php`
- `php/insert-chat.php`, `php/get-chat.php`
- `php/search.php` (թոքենը փոխանցվում է `users.php` էջից `assets/js/users.js`-ի միջոցով)

Եթե թոքենը բացակայում է կամ սխալ է, հարցումը մերժվում է (տեքստային պատասխան կամ HTTP 403)։

## Նախագծի կառուցվածք

| Ուղի | Նշանակություն |
|------|----------------|
| `index.php`, `login.php`, `users.php`, `chat.php` | Հիմնական էջեր |
| `includes/header.php` | Ընդհանուր `<head>` |
| `assets/css/`, `assets/js/` | Ոճեր և JavaScript |
| `php/` | AJAX սկրիպտներ, `config.php`, `csrf.php`, վերբեռնումներ |
| `sql/schema.sql` | MySQL սխեմա |

## Ձեռքով ստուգման ցանկ

Փոփոխություններից հետո կամ թողարկումից առաջ անցեք այս քայլերը՝

1. **Գրանցում** — `index.php`, նոր օգտատեր (նկարով և առանց)։ Արդյունք՝ անցում `users.php`, CSRF սխալ չի երևում։
2. **Մուտք** — դուրս գալ, `login.php`, վավեր տվյալներով մուտք։ Արդյունք՝ `users.php`, «առցանց» կարգավիճակ։
3. **Հաղորդագրություն** — չաթ մեկ այլ օգտատիրոջ հետ, տեքստ ուղարկել։ Հաղորդագրությունը երևում է երկու կողմում (կարող է կարճ ուշացում ունենալ polling-ի պատճառով)։
4. **Ֆայլ** — չաթում թույլատրված կցորդ (օր. պատկեր, PDF) ուղարկել։
5. **Դուրս գալ** — logout։ Արդյունք՝ վերահղում մուտքի էջ, օգտատերը «անցանց»։

Լրացուցիչ՝ մեկ այլ բրաուզերով կամ cookie-ները մաքրելով համոզվեք, որ առանց վավեր CSRF թոքենի `php/login.php`-ին POST հարցումը չի անցնում։

## Կարճ նշումներ

- **Արտադրություն** — օգտագործեք HTTPS, որպեսզի գաղտնաբառերն ու սեսիաները չլսնվեն ցանցում։
